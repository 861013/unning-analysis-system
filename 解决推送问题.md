# 解决GitHub推送问题

## 问题1：SSL证书验证失败（已解决）

已通过以下命令解决：
```bash
git config --global http.sslVerify false
```

## 问题2：身份验证问题

现在需要解决身份验证问题。有以下几种方案：

---

## 方案一：在URL中包含Personal Access Token（推荐）

### 步骤1：创建Personal Access Token

1. 访问 https://github.com/settings/tokens
2. 点击 **"Generate new token"** → **"Generate new token (classic)"**
3. 填写信息：
   - **Note**: 例如 `本地开发`
   - **Expiration**: 选择过期时间（建议90天或更长）
   - **Select scopes**: 至少勾选 `repo`（完整仓库访问权限）
4. 点击 **"Generate token"**
5. **立即复制生成的token**（只显示一次）

### 步骤2：更新远程仓库URL（包含token）

```bash
# 将YOUR_TOKEN替换为您刚才复制的token
git remote set-url origin https://YOUR_TOKEN@github.com/861013/unning-analysis-system.git
```

或者使用用户名和token：
```bash
git remote set-url origin https://861013:YOUR_TOKEN@github.com/861013/unning-analysis-system.git
```

### 步骤3：推送

```bash
git push -u origin main
```

---

## 方案二：使用Git凭据助手

### 步骤1：配置Git凭据助手

```bash
git config --global credential.helper store
```

### 步骤2：推送（会提示输入用户名和密码）

```bash
git push -u origin main
```

当提示时：
- **Username**: `861013`
- **Password**: 粘贴您的Personal Access Token（不是GitHub密码）

Git会保存凭据，下次推送时不需要再输入。

---

## 方案三：使用SSH方式（最安全，推荐长期使用）

### 步骤1：生成SSH密钥（如果还没有）

```bash
ssh-keygen -t ed25519 -C "1076359033@qq.com"
```

按提示操作（可以直接回车使用默认设置）

### 步骤2：查看公钥

```bash
cat ~/.ssh/id_ed25519.pub
```

复制输出的公钥内容

### 步骤3：添加到GitHub

1. 访问 https://github.com/settings/keys
2. 点击 **"New SSH key"**
3. 填写信息：
   - **Title**: 例如 `Ubuntu开发机`
   - **Key**: 粘贴刚才复制的公钥
4. 点击 **"Add SSH key"**

### 步骤4：更改远程仓库URL为SSH

```bash
git remote set-url origin git@github.com:861013/unning-analysis-system.git
```

### 步骤5：推送

```bash
git push -u origin main
```

## 注意事项

1. **Personal Access Token** 只显示一次，请妥善保存
2. 如果使用方案一（URL包含token），token会保存在Git配置中，注意安全
3. 方案三（SSH）是最安全的方式，推荐长期使用
4. 如果token过期，需要重新创建并更新
